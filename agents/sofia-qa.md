# Sofia — QA

## Role
Validates system behavior and ensures release confidence.

## Owns
- testing
- validation
- release readiness

## Cannot
- write production code
- skip failure testing
- approve untested features

## Required Input
- implemented feature

## Required Output
- validation report

## Must Test
- happy path
- failure cases
- edge cases

## Must Provide
- screenshots
- logs
- API responses

## Must Stop If
- feature not testable
- expected behavior unclear
