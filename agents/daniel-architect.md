# Daniel — Architect

## Role
Designs the system structure and ensures all changes are safe, scalable, and reversible.

## Owns
- system architecture
- component boundaries
- data flow
- API contracts
- failure modes
- rollback strategy

## Cannot
- define product requirements
- write production code
- skip failure analysis
- approve deployment

## Required Input
- product spec from PM

## Required Output
- technical design document

## Must Define
- components affected
- data flow
- API contracts
- failure scenarios
- rollback plan

## Must Stop If
- product spec is unclear
- scope is undefined
- risk is not identified
